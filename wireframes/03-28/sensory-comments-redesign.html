Sensory Comments — Design Proposals

Three approaches to improving how sensory evaluation comments are organized, labeled, and displayed. All designs group comments by sensory category and clearly indicate their evaluator source.

Option A Recommended

Category-Grouped Comments Card

Comments are grouped under their sensory category (Taste & Flavor, Texture, Appearance) with colored accents matching the category. Each comment clearly shows the metric it relates to and which evaluation it came from. Simple and clean — no tabs or filters needed.

Design rationale: Grouping by category creates a natural visual hierarchy that matches the scoring system (Taste, Texture, Appearance). The colored accent bars provide instant category recognition without needing to read labels. Source is shown inline with a colored dot matching the evaluator's chart color — this creates a direct visual link between comments and chart data.

Strengths

  • Scannable — users can jump to the category they care about
  • Zero interaction required to see all comments
  • Category colors match chart scoring categories
  • Evaluator source is always visible
  • Simple to implement — just re-groups existing data

Trade-offs

  • Card can get tall with many comments across categories
  • Comments are separate from the chart (scroll required)

Option B

Filterable Comments with Source Chips

Adds a filter row to quickly view comments from a specific evaluator or category. Useful when trials have many evaluations from multiple panelists and users want to compare feedback per-source.

Strengths

  • Quick per-evaluator comparison
  • Still shows category via accent bar color

Trade-offs

  • Filter row adds complexity for trials with few comments
  • More interactive — requires clicking to get value
  • May feel over-engineered for the common case (2–3 comments)

Option C

Comments Integrated into Chart View

Comments appear directly below the breakdown chart inside the same card, grouped by category. This eliminates the separate CommentsCard entirely and lets users see scores and feedback in one place. The chart rows that have comments show a subtle indicator icon.

Strengths

Trade-offs

  • Makes the analysis card significantly taller

  • More complex implementation — chart + comments in same scroll area
  • The fixed 480px height of the current card would need to become flexible

Recommendation

Option A — Category-Grouped Comments Card

Option A best balances the three goals:

  • Labeled by category: Category group headers with colored dots provide instant recognition
  • Clear source: Each comment shows evaluator name + log context with a color-coded dot
  • Full card width: Comments fill the card naturally — no cramped layout
  • Aesthetically clean: Matches the app's Stripe-like minimal style
  • Simplest to build: Re-groups existing data with no new interactive paradigms

Option B's filters add complexity that most trials won't need (typically 2–4 comments). Option C is appealing but scoping comments to the selected log loses the cross-log view, and expanding the card height disrupts the current 480px photo+scores grid.

If we later find users frequently want to cross-reference a specific comment with its chart bar, we can add the 💬 indicator icons from Option C to the breakdown chart as a small enhancement on top of Option A.
