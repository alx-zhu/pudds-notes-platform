Sensory Comments v2 — Wide-Screen Designs

Addresses two problems: (1) comments must fill the wide right column naturally instead of stretching a narrow list, and (2) avoid dark/muted card backgrounds per comment that create visual heaviness. All designs use a white card surface with structure provided by layout, typography, and subtle borders only.

Option A Recommended

Category Columns

Three equal columns — one per sensory category — divided by subtle vertical rules. Comments appear as threaded messages under their metric name, showing the evaluator's avatar, name, and log context. Fills wide screens naturally.

Inspired by: Slack's channel layout, Linear's sidebar columns, Trello's column structure

Why this works on wide screens: Three columns divide the space naturally and create a clear information hierarchy. Each category owns its column — users scan left-to-right by domain. The column layout mirrors how food scientists actually think: "How's the taste? How's the texture? How does it look?" No background fills on individual comments — structure comes from layout, metric names, and divider lines, keeping the surface clean and light. Evaluator identity is shown via avatar + name (Slack pattern), which is faster to scan than inline text labels.

Strengths

  • Fills wide screens naturally — no empty space
  • Clean white surface — no dark card backgrounds
  • Category scan pattern matches how scientists think
  • Evaluator avatars make source instantly recognizable
  • Metric grouping within columns adds further structure

Trade-offs

  • Columns can be uneven if most comments are Taste
  • Narrower per-column width on medium screens

Option B

Metric Threads, 2-Column Flow

Comments are grouped by metric, and metrics flow into a 2-column grid under their category heading. Each metric acts as a mini thread. A small colored accent line identifies the category. On narrower screens, collapses to single column.

Inspired by: Linear's activity feed, GitHub PR conversations, Notion's database views

Why this works: The 2-column grid fills width while keeping each comment at a comfortable reading width (~350–450px). Metric threads create natural grouping — if two evaluators commented on the same metric, their messages stack in the same thread. The small accent line is the only color, keeping the design light.

Strengths

  • Good width utilization via 2-column grid
  • Thread pattern handles multiple comments per metric naturally
  • Gracefully collapses to 1-column on narrower screens
  • Minimal color use — clean and quiet

Trade-offs

  • Empty grid cells when odd number of metric threads
  • Category boundaries less visually distinct than columns
  • More vertical space than Option A

Option C

Compact Comment Cards Grid

Each comment is a lightweight bordered card in an auto-filling grid. Cards self-organize into rows, adapting to screen width. Category labels act as section dividers. The most compact option — best when there are many comments.

Inspired by: Notion's gallery view, Attio's card-based records, Apple Notes grid

Recommendation

Option A — Category Columns

Option A is the strongest choice because it solves the width problem structurally — three columns fill a wide screen by design rather than by stretching content. The fixed column-per-category layout also maps to the scoring system's three dimensions, creating a consistent mental model across charts and comments.

Key design decisions:

  • No background fills on comments — structure comes from columns, dividers, and typography
  • Evaluator avatars (Slack pattern) instead of inline text labels — faster visual parsing
  • Metric names as thread headers — if two people comment on Sweetness, their messages stack naturally
  • Colored column header underline — the only deliberate color, tying back to the summary chart bars

Option B is a solid alternative if we want a more compact layout that handles uneven comment distribution better. Option C works best at scale (8+ comments) but loses the threaded structure that makes multi-evaluator feedback readable.
