Design Decisions (v2)

  • Cards use min-width 280px. auto-fill, minmax(280px, 1fr) ensures cards never compress too small — grid wraps to fewer columns instead.
  • Formulation moved below image. Trial number stands alone at top for instant identification. Image has fixed aspect-ratio (16:9) that cannot be affected by content above or below.
  • Formulation is plain text, not colored pills. Reduces visual noise — the abbreviation is bold, percentage is muted. Slash separators keep it readable. Eliminates the many-colored-badges problem.
  • Table: scores in one cell, all labeled. Overall, Taste, Texture, Color shown inline with labels — including "Overall" which was previously unlabeled. Compact but every number is explicitly identified.
  • Table: metadata split into individual columns. Date, Processing, and Flavor each have their own column rather than being combined into one "Metadata" cell. Non-technical users can read each field independently.
  • Card empty state matches real layout. When no evaluations exist, the score area shows the same overall + category bar structure with em-dash placeholders and muted styling. Card height stays consistent regardless of score state.
  • Reduced color palette on cards. Only the metadata badges (processing + flavor) use colored backgrounds. Formulation and scores use text color/weight hierarchy instead — much less visual clutter.
